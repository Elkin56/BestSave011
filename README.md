# BestSave — пробная версия (Vercel)

Бесплатный триал: бот-архиватор Telegram-чатов + Mini App для просмотра архива.
Пользователь добавляет бота в свои группы/каналы, назначает администратором —
бот сохраняет сообщения, ловит правки и хранит копии удалённого.

---

## ⚠️ Что этот продукт МОЖЕТ и чего НЕ может

Это не маркетинг, а границы платформы Telegram. Понимание сэкономит вам недели.

**Может (легально, без риска бана):**
- Архивировать сообщения в группах/каналах, где бот — **администратор**
- Ловить **редактирования** (`edited_message` боту приходит) и хранить версию «до»
- Хранить копии, чтобы показать удалённое **позже**
- Сохранять медиа (фото, видео, голосовые, файлы)

**Не может (ограничение Telegram, обойти нельзя без нарушения ToS и закона):**
- Видеть **личные 1-на-1 чаты** пользователя с другими людьми.
  Bot API отдаёт боту только то, что адресовано самому боту или отправлено
  в чат с ботом. Чужую личную переписку бот не видит — и не должен.
- Получать апдейт **в момент удаления**. У ботов такого события нет.
  Поэтому мы не «детектируем удаление», а **храним то, что видели раньше**.

Обойти это можно только userbot-ом (вход в аккаунт по номеру телефона) — а это
прямое нарушение Telegram ToS (бан аккаунта), передача чужого аккаунта третьей
стороне и, в большинстве юрисдикций, незаконный перехват чужих сообщений.
Здесь такого нет намеренно.

## Если Vercel пишет «unmatched-function-pattern» или не видит api

Это значит, что **папка `api` не загрузилась в репозиторий** (частая беда при
загрузке через веб-GitHub — вложенные папки теряются). Проверьте на GitHub, что
в репозитории есть все четыре файла: `api/bot.js`, `api/health.js`,
`api/chats.js`, `api/messages.js`. Если их нет — загрузите заново.

Надёжнее не тащить папки мышкой в браузере, а через командную строку:
```bash
cd bestsave-trial
git init && git add -A && git commit -m "bestsave trial"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПО.git
git push -u origin main
```
`git add -A` берёт все файлы и папки рекурсивно — ничего не потеряется.

В этой версии структура `api/` **плоская** (без вложенных папок) именно затем,
чтобы веб-загрузка на GitHub её не теряла.

## Развёртывание — по шагам

### 1. Создать бота
В Telegram напишите [@BotFather](https://t.me/BotFather): `/newbot`.
Сохраните **BOT_TOKEN**. Затем **обязательно** выключите Privacy Mode, иначе
бот не увидит сообщения в группах даже будучи админом:
`/setprivacy` → выберите бота → **Disable**.


### 2. Завести базу Postgres
Любой управляемый Postgres. Проще всего — [Neon](https://neon.tech) (free tier)
или Vercel Postgres. Скопируйте строку подключения (`DATABASE_URL`).

### 3. Задеплоить на Vercel
```bash
npm i -g vercel
cd bestsave-trial
vercel            # первый деплой (preview)
vercel --prod     # прод
```
Либо загрузите папку в Git и импортируйте репозиторий в дашборде Vercel.

### 4. Переменные окружения (Vercel → Settings → Environment Variables)
| Переменная | Значение |
|---|---|
| `BOT_TOKEN` | токен от BotFather |
| `DATABASE_URL` | строка подключения Postgres |
| `WEBHOOK_SECRET` | любая случайная строка (защита вебхука) |
| `ADMIN_TG_IDS` | ваш tg id (доступ к панели владельца; гейт на вас не действует) |
| `COMMUNITY_CHAT` | `@юзернейм` канала сообщества, по умолчанию `@bestsavee` |
| `COMMUNITY_TITLE` | название канала в интерфейсе, по умолчанию `BestSave Community` |
| `COMMUNITY_URL` | только для приватного канала: инвайт-ссылка |
| `REQUIRED_INVITES` | сколько друзей нужно привести, по умолчанию `3` |

После добавления сделайте **Redeploy**, чтобы переменные подхватились.

### 5. Инициализировать схему
Откройте `https://ваш-проект.vercel.app/api/health` — таблицы создадутся,
в ответе увидите `"schema": true`.

### 6. Привязать вебхук
```bash
BOT_TOKEN=... \
WEBHOOK_URL=https://ваш-проект.vercel.app/api/bot \
WEBHOOK_SECRET=...тот же секрет... \
node scripts/set-webhook.js
```
Ответ `{"ok": true}` — вебхук стоит.

### 7. Подключить Mini App к боту
В BotFather: `/newapp` (или `/myapps`) → выберите бота → укажите URL
`https://ваш-проект.vercel.app`. Появится кнопка запуска приложения.

### 8. Проверить
1. Создайте тестовую группу, добавьте бота, назначьте админом
2. Бот пришлёт «Готово, архив начат»
3. Напишите пару сообщений, одно отредактируйте
4. Откройте Mini App → чат появится, правка сохранится

## Условия доступа (гейт)

Перед первым использованием пользователь должен выполнить два условия:
подписаться на канал сообщества и привести `REQUIRED_INVITES` друзей по
личной ссылке `t.me/<бот>?start=ref<id>`.

**Бота нужно сделать администратором канала.** Без этого `getChatMember`
не отвечает, и подписка у всех повиснет в состоянии «проверить не удалось»
— гейт не пустит никого.

Как это устроено:

* Проверка стоит в `api/[...route].js` (набор `GATED`), а не в каждом
  обработчике — новый эндпоинт нельзя забыть закрыть. Отказ приходит как
  `403 { error: 'gate_required', gate: {...} }`, состояние едет прямо в ответе,
  и фронтенд рисует экран условий вместо голой ошибки.
* Выполненный гейт фиксируется в `app_user.gate_passed_at` и больше не
  пересматривается. Отписка от канала или сбой Telegram API не отбирают
  доступ к уже собранному архиву — данные принадлежат человеку.
* `referral.invited_tg_id` — первичный ключ: один человек засчитывается
  ровно одному пригласившему и ровно один раз. Реферал считается только за
  нового пользователя (проверка `userExists` идёт **до** `upsertUser`),
  самоприглашение отсекается, при недоступной базе не засчитывается ничего.
* Владельцы из `ADMIN_TG_IDS` гейт не проходят.
* Архивация во время гейта не останавливается: чаты, куда бот уже добавлен,
  продолжают сохраняться. Закрыт только просмотр.

Команда `/invite` в боте показывает статус и личную ссылку.

⚠️ После обновления **переустановите вебхук** (`/api/setup?secret=…`):
в `allowed_updates` добавился `callback_query`, без него кнопка «Проверить»
под экраном условий молча не работает.

## Модель безопасности

- **Каждый** `/api/*`-запрос несёт `initData` в заголовке `Authorization: tma <...>`.
  Сервер проверяет HMAC-подпись **и возраст** (не старше суток, не из будущего).
  Без валидной подписи — 401.
- `/api/messages` дополнительно проверяет, что запрошенный чат **привязан именно
  этим пользователем** — иначе по chatId можно было бы читать чужой архив (403).
- Вебхук бота защищён `secret_token`: Telegram шлёт его в заголовке, чужой POST
  на `/api/bot` отклоняется.
- Заголовки безопасности (`X-Content-Type-Options`, `Referrer-Policy`) в `vercel.json`.

## Известные ограничения триала
- Serverless-функции живут секунды: тяжёлый бэкфилл истории здесь неуместен,
  бот архивирует «с момента подключения», а не задним числом.
- `ensureSchema()` вызывается на месте (bootstrap). В проде — заменить на миграции.
- Медиа сохраняются как факт (тип + метаданные); сами файлы в S3 — задача платной версии.
